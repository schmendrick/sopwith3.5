This file contains a description of what is in the sprite file and is used
by mkg.c (along with sopwith.spr) to generate the C files which are linked into
the executable.

The left column is the actual data and the right column is the description of
what the data means.

The data is in the following format:
4 numbers: colour data (possible bytes in the sprite file)
string   : input filename (sprite file)
string   : output filename (sprites)
string   : output filename (collision data)
number   : number of records to follow. Each record consists of:
  flag      : 0 if no collision data is required.
              1 if one of the dimensions (the most significant one) is colour,
              2 if the sprite is neutral (no colour dimension)
  string    : name of set of sprites
  number (n): number of dimensions to follow
  n numbers : dimensions (most significant first)
  number    : vertical dimension in pixels
  number    : horizontal dimension in pixels
